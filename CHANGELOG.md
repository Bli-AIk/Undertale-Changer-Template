## v1.0.2 - Preface

First, I made a big mistake—since UCT updated to v1.0.2, there have been many updates, but its version number has remained at v1.0.2. This results from neglecting the update log for a long time.

In fact, updating UCT to v1.0.0 was somewhat of a mistake—at that time, I thought my template was already complete, but now it seems I still have a long way to go. Nevertheless, changing the version number back would be more troublesome. So, let’s leave it as is for now.

In the last update of v1.0.2, I added this file—**CHANGELOG.md**—to record the contents of each update and plans for the next update.

The version number updates will follow these rules:

1. When a **PATCH** update occurs (the **X** in 1.0.X), it generally means this update is a bug fix or a modification/refactoring of existing features without adding new functionality or content.
2. When a **MINOR** update occurs (the **X** in 1.X.0), it usually means this update adds new functionality or content.
3. When a **MAJOR** update occurs (the **X** in X.0.0), it generally means… <small>*Alright, I estimate that MAJOR will never change.*</small>
4. Before updating to the next version, the version number will have a suffix -part X. X represents the number of completed task items.

The update format is as follows:

#### vX.X.0 -> vX.X.1 Summary of Updates

- [x] Task 1 (Completed)
- [x] Task 2 (Completed)

Plan for XX/XX/XX

Completion on XX/XX/XX

#### vX.X.1 -> vX.X.2 Summary of Updates

- [ ] Task 1 (Not Completed)
- [ ] Task 2 (Not Completed)

Plan for XX/XX/XX (x-x tasks)

Completion on XX/XX/XX

The completion time refers to the time when the last item is completed, and that's it.
